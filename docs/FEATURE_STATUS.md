## Edge cases observed

- **Timer at 0:00** — `tick()` checks `remainingSec <= 1` *before* decrementing, so it transitions cleanly without ever displaying `-1` (`focusStore.ts:151`).
- **Refresh mid-session** — the persisted snapshot does not include `status` / `remainingSec`, and the custom `merge` forces `status = "idle"` and `remainingSec = durationSec`. No stale countdown resumes. Verified by reading `focusStore.ts:211-236`.
- **Dismissed reflection** — still adds the session to `sessionLog` with `reflection: null` (`focusStore.ts:195-205`), so closing the modal still records that the session happened.
- **Default boot status** — store creator sets `status: "running"` (`focusStore.ts:117`), but the `merge` function overrides it to `idle` even when there is no persisted state (because `persistedState ?? {}` makes the spread a no-op and the explicit `status: "idle"` always wins). Confusing but correct in effect. See `CODE_QUALITY_REVIEW.md`.
- **Daily plan duration overrides timer length** — `setDailyPlan` rewrites `durationSec` based on `plan.plannedDurationMin` (`focusStore.ts:167-180`). If the user is currently running/paused, `remainingSec` is preserved so the active session isn't disrupted.
